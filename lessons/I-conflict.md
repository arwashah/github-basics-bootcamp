# Lesson I: Resolve a merge conflict

Conflicts happen when two branches change the same lines.

What it is:
- A merge conflict means Git cannot auto-combine changes to the same area.

Why it is useful:
- Forces a deliberate choice so you do not lose important edits.
- Builds confidence in safely integrating parallel work.

Checklist mapping:
- I) Resolve a merge conflict

Tip:
- Follow `conflict-exercise/README.md` for a safe practice conflict.
