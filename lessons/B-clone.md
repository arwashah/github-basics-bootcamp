# Lesson B: Clone locally

Cloning brings the repo to your machine so you can work with Git.

What it is:
- A clone is a full local copy of the repository, including its history.

Why it is useful:
- Lets you edit files with local tools and work offline.
- Keeps a full history so you can inspect or revert changes.

Checklist mapping:
- B) Clone locally

Tip:
- Use the HTTPS URL if you are new to SSH.
