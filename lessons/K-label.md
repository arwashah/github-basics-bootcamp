# Lesson K: Add a label to the Issue

Labels help categorize and filter issues.

Objectives:
- Apply a label to an issue.
- Understand how labels support triage and reporting.

Key concepts:
- A label is a tag you apply to issues or pull requests.
- Labels can be reused across many items for quick filtering.

Steps:
1. Open an issue in GitHub.
2. In the right sidebar, find the "Labels" section.
3. Select an existing label or click "Edit labels".
4. If needed, click "Create new label".
5. Choose a clear name and color, then save.

Verify:
- The issue displays the label near the title.
- The label appears in the repository label list.

Common pitfalls:
- Using inconsistent label names (case or spacing changes).
- Creating too many overlapping labels.

Checklist mapping:
- K) Add a label to the Issue

Tip:
- Create a new label if none exist.
